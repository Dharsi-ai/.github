# Security Policy

## Reporting a Vulnerability

Dharsi takes the security of our products and services seriously. If you believe you have found a security vulnerability in any Dharsi-owned repository, please report it to us responsibly.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

Email your findings to: **security@dharsi.ai**

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested remediation (optional)

### What to Expect

- **Acknowledgement** within 2 business days
- **Initial assessment** within 5 business days
- We will keep you informed of our progress toward resolution
- We will credit you (if desired) once the issue is resolved

### Scope

This policy applies to all repositories under the [Dharsi-ai](https://github.com/Dharsi-ai) GitHub organisation.

### Out of Scope

- Social engineering attacks
- Denial of service attacks
- Issues in third-party dependencies (please report these to the respective maintainers)

## Supported Versions

Only the latest release of each product is supported with security updates.

## Security Practices

- All private repositories require signed commits from authorised contributors
- Dependabot alerts and secret scanning are enabled organisation-wide
- Two-factor authentication is mandatory for all organisation members

---

Thank you for helping keep Dharsi and our users safe.
