# Contributing to Dharsi

Thank you for your interest in Dharsi's repositories.

## Public Repositories

For our public repositories (such as `dharsi-docs`), we welcome:

- **Bug reports** — If you spot an error, please [open an issue](https://github.com/Dharsi-ai/dharsi-docs/issues)
- **Suggestions** — Ideas for new content or improvements are appreciated

At this time, we do not accept external pull requests on public repositories. All content is authored and maintained by the Dharsi team.

## Private Repositories

Private repositories are accessible only to authorised Dharsi team members and development partners. If you are a contributor on a private repository:

### Branch Naming

Use the following conventions:

- `feature/short-description` — New features
- `fix/short-description` — Bug fixes
- `docs/short-description` — Documentation updates
- `chore/short-description` — Maintenance and tooling

### Commit Messages

Write clear, concise commit messages:

```
type: short description

Optional longer explanation if needed.
```

Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `ci`

### Pull Requests

- All changes require a pull request — no direct commits to `main`
- PRs require at least one review before merging
- Keep PRs focused — one feature or fix per PR
- Link relevant issues in the PR description

### Code Standards

- Follow the linting and formatting rules defined in each repository
- Include appropriate tests for new functionality
- Update documentation when changing behaviour

## Questions

If you have questions about contributing, reach out to **info@dharsi.ai**.
